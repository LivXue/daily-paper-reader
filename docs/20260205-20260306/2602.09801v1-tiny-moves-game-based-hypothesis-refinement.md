## 摘要
大多数用于科学发现的机器学习方法将假设视为端到端的预测，从而掩盖了科学推理的增量结构。我们提出了“假设博弈”（The Hypothesis Game），这是一种用于假设细化的符号形式化方法，其中大语言模型（LLM）智能体使用固定的推理动作语法在共享的假设状态上进行操作。该框架的动机源于以下观察：科学进步通常是通过基于领域背景的小型局部修订，而非大规模重写来实现的。我们利用 LLM 智能体实例化了一个最小博弈，并在通路级别的机制细化任务上对其进行了评估。在损坏恢复（corruption recovery）的主要设置中（即假设包含受控错误），基于博弈的方法比强提示词基准能更一致地消除更多错误并获得更高的精确度，同时通过增量编辑保留了有效的结构。在基于部分线索的次要重建设置中，其表现与最强基准相当，这表明即使在基准真相（ground-truth）恢复困难的情况下，显式的基于动作的细化仍然具有竞争力。这些发现支持将基于博弈的推理作为一种原则性路径，以构建更具可控性、可解释性和可迁移性的科学发现假设细化系统。

## Abstract
Most machine learning approaches to scientific discovery frame hypotheses as end-to-end predictions, obscuring the incremental structure of scientific reasoning. We propose The Hypothesis Game, a symbolic formalism for hypothesis refinement in which LLM agents operate on a shared hypothesis state using a fixed grammar of reasoning moves. The framework is motivated by the observation that scientific progress often proceeds through small, localized revisions, grounded in domain context, rather than extensive rewrites. We instantiate a minimal game with LLM agents and evaluate it on pathway-level mechanistic refinement tasks. In the primary setting of corruption recovery, where hypotheses contain controlled errors, the game-based approach consistently removes more errors and achieves higher precision than strong prompting baselines, while preserving valid structure through incremental edits. In a secondary reconstruction setting from partial cues, it performs comparably to the strongest baseline, indicating that explicit move-based refinement remains competitive even when ground-truth recovery is difficult. These findings support game-based reasoning as a principled route to more controllable, interpretable, and transferable hypothesis refinement systems for scientific discovery.

### 2. 方法论：假设博弈框架
*   **核心思想**：将假设细化形式化为一个符号博弈过程。LLM 智能体作为“玩家”，在一个共享的假设状态上操作，且必须遵守一套固定的“推理语法”（动作集）。
*   **关键技术细节**：
    *   **假设表示**：假设被表示为一组片段（Fragments），可以是结构化文本或三元组（主体-关系-客体）。
    *   **推理语法（动作集）**：
        1.  **Prune（修剪）**：移除假设中薄弱或错误的组件。
        2.  **Expand（扩张）**：通过检索外部文献库（Corpus）或利用 LLM 内部知识增加新的连接。
        3.  **Debate（辩论）**：多个智能体针对特定争议点进行多轮讨论并达成共识。
    *   **游戏控制器（Game Master）**：由 LLM 担任，负责诊断当前假设状态、根据策略选择下一个动作，并决定何时终止博弈。
    *   **博弈模式**：通过提示词设定不同的模式（如“发现模式”倾向于扩张，“验证模式”倾向于修剪）。

### 6. 主要结论与发现
*   **损坏恢复表现优异**：在错误修复任务中，Hypothesis Game 在错误移除率、精确度（Precision）和 F1 分数上显著优于 ReAct 和 CoT。它能精准定位并剔除错误，同时保留正确的结构。
*   **重建任务具有挑战性**：在从零重建任务中，该方法与最强的 ReAct 基准表现相当。虽然精确度较高，但所有方法在完全恢复复杂生化通路方面仍面临困难。
*   **增量编辑的优势**：定性分析显示，该方法倾向于进行局部、可追溯的修改，而基准方法往往会产生大幅度的文本波动，导致“改对一个地方，改错三个地方”。

### 7. 优点与亮点
*   **高可解释性**：每一步修改都有明确的动作标签（如“因为证据不足而修剪”），形成了清晰的推理轨迹。
*   **细粒度控制**：通过限制动作语法，防止了 LLM 常见的“幻觉式重写”，使假设演进更加稳健。
*   **模块化设计**：推理语法和控制器是解耦的，未来可以轻松引入更复杂的评分函数或强化学习控制器。

### 8. 不足与局限
*   **重建任务瓶颈**：在缺乏足够先验信息时，增量式细化可能陷入局部最优，难以实现跨越式的发现。
*   **依赖控制器质量**：系统的表现高度依赖于“Game Master”诊断错误的准确性，如果 LLM 无法识别细微的生物学错误，后续动作将失效。
*   **表示形式限制**：目前主要处理文本和简单的三元组，对于复杂的空间结构或动态动力学模拟的集成尚在概念阶段。
*   **成本问题**：多轮博弈和多智能体辩论会显著增加 API 调用成本和推理延迟。
